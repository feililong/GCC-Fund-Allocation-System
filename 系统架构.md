核验 Agent (Verifier)：监控 GitHub PR、Release、链上事件，自动验证里程碑完成状态
执行 Agent (Executor)：验证通过后自动触发 Gnosis Safe 多签交易
通知 Agent (Notifier)：多渠道同步状态，支持 Telegram、Discord、邮件
审计 Agent (Auditor)：生成审计报告，更新公开仪表盘

# 系统架构设计

## 概览

GCC 公共资金分配自动化系统采用微服务架构，由四个核心 Agent 组成，通过 Redis 消息队列通信，所有状态持久化至 PostgreSQL 和 IPFS。

## 核心组件

**职责**：监控外部数据源，自动验证里程碑完成状态

- GitHub API（PR、Release、Commit、Code Coverage）
- 链上事件（交易确认、合约事件日志）
- IPFS/Arweave（文档存证）
