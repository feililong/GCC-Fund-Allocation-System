# 系统架构设计

## 概览

GCC 公共资金分配自动化系统采用微服务架构，由四个核心 Agent 组成，通过 Redis 消息队列通信，所有状态持久化至 PostgreSQL 和 IPFS。

## 核心组件

### 核验 Agent (Verifier Agent)

**职责**：监控外部数据源，自动验证里程碑完成状态

**数据源**：
- GitHub API（PR、Release、Commit、Code Coverage）
- 链上事件（交易确认、合约事件日志）
- IPFS/Arweave（文档存证）
监听里程碑提交事件（GitHub Issue 或 API 调用）
并行查询多个数据源（GitHub + 链上 + IPFS）
应用验证规则（配置化权重）
计算置信度评分（0-100）
阈值判断：
≥ 85分：自动通过
70-84分：转人工复核
< 70分：拒绝并通知

### 2. 执行 Agent (Executor Agent)

**职责**：验证通过后，自动触发资金拨付

**工作流**：
接收 milestone.verified 事件
验证项目白名单和金额限制
构造 Gnosis Safe 多签交易
预估 Gas 费用（动态调整）
提交交易至内存池
监听交易确认（最多等待 5 分钟）
更新状态并触发 notification.sent 事件

- 短信（Twilio，可选）
